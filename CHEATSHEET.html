Local AI / macOS / Apple Silicon

Шпаргалка по локальному AI-стеку

Готовый быстрый лист для ежедневной работы с llamafile, локальным UI, OpenAI-compatible API и переключением профилей модели.

UI
http://127.0.0.1:8080

API
http://127.0.0.1:8080/v1

Быстрый старт

cd ~/ai/llamafile-local
./llm.sh start
./llm.sh open
Основной профиль: 7B UI в браузере API на localhost

Проверка стенда

./llm.sh status
./llm.sh health
./llm.sh test "Проверь локальный стенд"

Быстро показывает, жив ли процесс, отвечает ли UI и доступен ли OpenAI-compatible API.

Профили модели

./llm.sh use-bootstrap
./llm.sh restart

Bootstrap: 1.5B, быстрее и легче.

./llm.sh use-7b
./llm.sh restart

Основной: 7B, лучший баланс под 24 ГБ RAM.

Загрузка моделей

./llm.sh download recommended

recommended = рабочий 7B

./download-model.sh bootstrap
./download-model.sh power
Загрузка умеет resume curl -C -

Python и агенты

python3 client.py "Сделай короткий ответ по-русски"
curl http://127.0.0.1:8080/v1/models

client.py автоматически подхватывает активный профиль из config.env.

Логи и остановка

./llm.sh logs
./llm.sh stop
Основной лог лежит в logs/llamafile.log. Если UI открылся, а ответов нет, начинайте отсюда.

Самый практичный сценарий

# открыть уже работающий стек
cd ~/ai/llamafile-local
./llm.sh open

# если нужно перезапустить
./llm.sh restart
./llm.sh open

# если хочется быстрый профиль
./llm.sh use-bootstrap
./llm.sh restart

# если нужен основной рабочий профиль
./llm.sh use-7b
./llm.sh restart
Файл можно открывать локально в браузере. Проект: ~/ai/llamafile-local
