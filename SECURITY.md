# Security Policy

## Supported Scope

Этот репозиторий в первую очередь является локальным self-hosted setup-проектом, а не production security framework.

Поэтому security-ожидания здесь такие:

- проект помогает поднять локальный LLM-стенд
- проект не обещает enterprise-grade security baseline
- локальные small/quantized модели не стоит считать безопасной основой для сложных автономных agent-сценариев

Особенно это важно для tool-using и agent-пайплайнов.

## Что считать security issue

Сюда стоит репортить:

- утечки чувствительных данных из bootstrap/logging flow
- опасное поведение скриптов, которое может привести к нежелательным локальным действиям
- ошибки конфигурации, которые по умолчанию делают локальный сервис небезопасно доступным извне
- небезопасные значения по умолчанию, если они реально повышают риск для пользователя

## Что не стоит считать security bug этого репозитория

- общие ограничения open-source моделей
- prompt injection как класс проблем у самих моделей
- ограничения safety у маленьких квантованных моделей
- уязвимости сторонних проектов и моделей, на которые этот репозиторий только ссылается

## How To Report

Если вы нашли потенциальную проблему безопасности:

1. Не публикуйте exploit details сразу в публичном issue.
2. Сначала откройте приватный контакт через GitHub owner/repo contact, если он доступен.
3. Если приватный канал недоступен, можно открыть issue без деталей и кратко указать, что проблема security-related.

## Operational Advice

Для безопасного локального использования:

- оставляйте `HOST=127.0.0.1`, если вам не нужен доступ из сети
- не открывайте локальный endpoint наружу без понимания последствий
- осторожно относитесь к agent-сценариям с tool use
- для серьезных интеграций перепроверяйте ограничения выбранной модели и рантайма
