# CONFIG.md

## Параметры запуска

Все основные параметры лежат в `config.env`.

### `HOST`

- По умолчанию: `127.0.0.1`
- Безопасный локальный режим.
- Меняйте на `0.0.0.0`, только если нужен доступ с других устройств в вашей сети.

### `PORT`

- По умолчанию: `8080`
- На этом порту работает и web UI, и OpenAI-compatible API.

### `MODEL_FILE`

- Путь к GGUF-модели.
- Для multipart GGUF обычно достаточно указывать первый файл, если остальные части лежат рядом.
- Переключать безопаснее через `./switch-model.sh`, а не ручным редактированием.

### `MODEL_ALIAS`

- Имя модели, которое будет видно в ответах API.
- Удобно для клиентского кода и будущих агентных обвязок.
- Тоже обновляется через `./switch-model.sh`.

## Готовые профили

### Bootstrap

- Команда переключения: `./switch-model.sh bootstrap`
- Модель: `Qwen2.5-1.5B-Instruct Q4_K_M`
- Когда использовать: первый запуск, быстрый smoke test, легкий интерактив

### Recommended

- Команда переключения: `./switch-model.sh recommended`
- Модель: `Qwen2.5-7B-Instruct Q4_K_M`
- Когда использовать: основной рабочий режим под 24 ГБ RAM

### Power

- Команда переключения: `./switch-model.sh power`
- Модель: `Qwen2.5-14B-Instruct Q4_K_M`
- Когда использовать: только если качество важнее скорости и вы готовы снижать контекст

### `CTX_SIZE`

- По умолчанию: `8192`
- Главный рычаг управления памятью.
- Чем больше контекст, тем выше расход памяти и время обработки.

### `THREADS`

- Число CPU-потоков для генерации.
- Практичный стартовый профиль: `8`.

### `THREADS_BATCH`

- Потоки для обработки входного промпта.
- Обычно можно держать равным `THREADS`.

### `BATCH_SIZE`

- По умолчанию: `512`
- Влияет на скорость обработки промпта и на расход памяти.
- Если появляются проблемы со стабильностью или памятью, уменьшайте до `256` или `128`.

### `GPU_LAYERS`

- По умолчанию: `999`
- На Apple Silicon `llamafile` использует Metal по умолчанию, поэтому этот параметр нужен, чтобы отдать как можно больше слоев на GPU.
- Если видите ошибки памяти, уменьшайте: `256`, `128`, `64`.

### `PARALLEL`

- По умолчанию: `1`
- Для 24 ГБ RAM безопасно начинать именно с единицы.
- Увеличение повышает давление на память.

### `TIMEOUT`

- HTTP timeout сервера в секундах.
- Для тяжелых локальных моделей `600` это нормальный безопасный дефолт.

### `EXTRA_ARGS`

- Сюда можно добавлять дополнительные флаги `llamafile`, если захотите поэкспериментировать.

## Управление памятью

### Если все работает стабильно

- Оставьте `CTX_SIZE=8192`
- Оставьте `PARALLEL=1`
- Не трогайте `GPU_LAYERS`

### Если модель падает или уходит в swap

Действуйте по порядку:

1. Уменьшите `CTX_SIZE` до `4096`
2. Уменьшите `BATCH_SIZE` до `256`
3. Уменьшите `GPU_LAYERS`
4. Оставьте активным `Qwen2.5-1.5B-Instruct Q4_K_M`

## Ускорение на Apple Silicon

- Держите модель в локальном SSD, не на внешнем медленном диске.
- Не повышайте `PARALLEL`, если нужен лучший отклик.
- Для интерактивного чата и первого старта `1.5B` заметно надежнее, чем тяжелый multipart-профиль.
- Для стабильной работы агентных PoC лучше сначала добиться устойчивости на `7B`, а уже потом пробовать `14B`.

## Когда уменьшать модель

Переходите на меньший профиль, если:

- важнее скорость, чем качество
- нужен быстрый cold start
- одновременно запущены другие тяжелые приложения
- появляется memory pressure в macOS
- нужен быстрый локальный smoke test

## Когда пробовать 14B

Имеет смысл только если:

- вы готовы к более медленному отклику
- вам реально нужно более высокое качество
- вы готовы снижать `CTX_SIZE`
- у вас нет параллельной тяжелой нагрузки на ноутбуке
