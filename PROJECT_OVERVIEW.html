Project Overview

Локальный AI-стек на llamafile

Практичный локальный стенд для open-source LLM на macOS Apple Silicon: браузерный UI, OpenAI-compatible API, быстрый bootstrap-профиль и основной рабочий профиль для чата, анализа текста, агентных PoC и локальных экспериментов.

Что это за проект

Это локальное окружение для запуска open-source языковых моделей без Docker, без оркестрации и без тяжелой инфраструктуры. Проект ориентирован на сценарий “открыл терминал, запустил 1–2 команды, открыл браузер и работаешь”.

Вся сборка организована вокруг llamafile, потому что он дает максимально простой путь к локальному запуску модели, web UI и API в одном компактном рантайме.

Базовый runtime проекта: Mozilla-Ocho/llamafile. Именно на его основе собраны локальный запуск модели, браузерный интерфейс и OpenAI-compatible endpoint.

Как он устроен

Бинарник bin/llamafile запускает модель, UI и API.
Модели models/ хранит GGUF-профили для быстрого и основного режима.
Управление start.sh, stop.sh, llm.sh, switch-model.sh.
Интеграция client.py и OpenAI-compatible API для будущих агентов и автоматизации.

Ключевые возможности

Локальный чат через браузер Можно открывать UI на http://127.0.0.1:8080 и работать без облака.
OpenAI-compatible API Подходит для Python-скриптов, локальных сервисов и агентных прототипов.
Быстрое переключение профилей Можно менять модель между 1.5B и 7B без ручной правки проекта.
Cloud fallback Есть шаблон окружения, чтобы позже подменить локальный endpoint на облачный.
Интеграция с OpenClaw и похожими системами Локальный /v1 endpoint можно использовать как backend для OpenAI-compatible agent PoC.

Почему этот стек полезен

Он закрывает самую частую практическую потребность: быстро иметь под рукой локальную модель для рабочих задач, не превращая ноутбук в инфраструктурный проект.

Это особенно удобно, когда нужен приватный локальный контур, когда хочется быстро проверить идею, или когда важно иметь предсказуемый стенд для PoC и агентных экспериментов.

Без облака по умолчанию Удобно для PoC Подходит для локальных агентов

Сценарии применения

1. Чат и повседневная работа Локальные вопросы, черновики, формулировки, быстрые идеи, без отправки данных наружу.
2. Анализ текста Суммаризация, выжимка, структурирование заметок, черновой разбор документов и длинных текстов.
3. Агентные PoC Тестирование локальных оркестраторов, prompt chains, CLI-агентов и tool-using сценариев.
4. OpenClaw как локальный agent frontend Можно подключить этот стек как self-hosted OpenAI-compatible provider и тестировать локальные agent workflows поверх вашего llamafile-endpoint.
5. Локальный API для своих приложений Можно подключать фронтенды, Python-сервисы, внутренние утилиты и playground-интерфейсы.
6. Резервный автономный режим Когда не хочется зависеть от облака, квот, сети или внешних API-изменений.
7. Безопасный bootstrap для экспериментов Можно быстро стартовать на 1.5B, а потом перейти на 7B, когда нужен лучший результат.

Текущий рабочий профиль

Сейчас основной профиль проекта — Qwen2.5-7B-Instruct Q4_K_M. Это основной баланс качества и скорости для машины с 24 ГБ RAM. 

./llm.sh use-7b
./llm.sh restart

Легкий fallback

Если нужен быстрый cold start, меньшее давление на память или оперативный smoke test, можно вернуться на Qwen2.5-1.5B-Instruct Q4_K_M. 

./llm.sh use-bootstrap
./llm.sh restart

Пример: OpenClaw + локальный endpoint

Этот проект можно использовать как локальный backend для OpenClaw, потому что llamafile поднимает OpenAI-compatible API на http://127.0.0.1:8080/v1.

Практически это означает: OpenClaw может ходить не в облачный провайдер, а в ваш локальный endpoint, если вы описываете кастомный OpenAI-compatible provider в своей конфигурации. 

{
  "agents": {
    "defaults": {
      "model": {
        "primary": "local/qwen2.5-7b-instruct-q4km"
      }
    }
  },
  "models": {
    "providers": {
      "local": {
        "type": "openai",
        "baseUrl": "http://127.0.0.1:8080/v1",
        "apiKey": "local"
      }
    }
  }
}

Такой сценарий особенно хорош для локальных agent PoC, тестирования prompts и автономных экспериментов.

При этом важно честно учитывать ограничения: OpenClaw в официальной документации предупреждает, что локальные small/quantized модели хуже с точки зрения safety и prompt injection, а для серьезных tool-using сценариев лучше использовать максимально сильные локальные модели и большой контекст. Поэтому здесь это стоит позиционировать как хороший локальный PoC-стек, а не как идеальную production-grade основу для сложных автономных агентов.

Что можно делать дальше

Подключить локальный стек к своим агентам Через client.py или напрямую через /v1/chat/completions.
Добавить собственные system prompts и шаблоны задач Для анализа документов, QA, прототипов ассистентов и внутренних инструментов.
Сделать cloud fallback без переписывания кода Шаблон уже лежит в cloud.env.example.
Расширить стек новыми GGUF-моделями Проект уже поддерживает сценарий аккуратного скачивания и переключения профиля.
Проект расположен в ~/ai/llamafile-local. Эту страницу можно открывать локально в браузере как обзор системы.
